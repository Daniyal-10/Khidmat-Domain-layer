# Programs: Operational Patterns

The Programs domain must flexibly support multiple distinct operational patterns common in the humanitarian, government, and NGO sectors globally:

## Pattern 1: Government Entitlement / Subsidy (DBT / Pensions)
**Characteristics:** Continuous intake, statutorily defined criteria, theoretically infinite capacity, heavily regulated.
**Examples:** Social pensions (widow, elderly, disability), food subsidies, Direct Benefit Transfers.
**Flow:** If criteria are met, enrollment is automatic. The cycle is continuous until demographic factors change (e.g., age out, income rise). Frequent compliance checkpoints (e.g., proof of life).

## Pattern 2: Phased Graduation / Livelihood Mission
**Characteristics:** Fixed intake window, defined cycle duration, structured multi-phase interventions.
**Examples:** Ultra-poor graduation models, rural livelihood missions, skill development initiatives.
**Flow:** Beneficiaries enter as a cohort. They receive consumption support, then asset transfers, then training. Requires strict milestone tracking and prerequisite rules to advance between phases.

## Pattern 3: Emergency / Disaster Activation
**Characteristics:** Trigger-based activation, rapid spin-up, relaxed or geography-only criteria, short duration.
**Examples:** Flood relief packages, conflict-zone emergency rations.
**Flow:** Program is created instantly via an emergency trigger. Humanitarian Overrides are frequently used. The program terminates fully or transitions into a "Recovery" successor program.

## Pattern 4: Multi-Tiered Geographic Rollout
**Characteristics:** A master program with localized variations.
**Examples:** A UN/WFP program adapted differently across countries; a Federal initiative implemented with variations by individual States.
**Flow:** The Parent Program sets the mandate and core funding. Child Programs inherit the structure but amend local eligibility criteria or implementing partners.

## Pattern 5: Earmarked NGO Sponsorship
**Characteristics:** 1:1 conceptual mapping, high donor compliance.
**Examples:** Child sponsorship, dedicated CSR trust funds.
**Flow:** Strict tracking of funding constraints. Case Management handles the individual plan, but the Program enforces strict audit checkpoints and reporting periods aligned with the donor's cycle.

## Pattern 6: Conditionality-Driven Assistance
**Characteristics:** Deliveries depend on external beneficiary behavior.
**Examples:** Cash conditional on maternal health visits or school attendance.
**Flow:** The Program defines the milestone constraint. The Case Plan pauses or executes based on Verification Operations confirming the condition.
