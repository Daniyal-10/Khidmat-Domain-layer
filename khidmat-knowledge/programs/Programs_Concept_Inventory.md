# Programs Concept Inventory

| Concept | Classification | Notes/Why |
|---|---|---|
| Program / Initiative | Type B (Programs) | The core structural entity (Pilot, Permanent, Emergency). |
| Program Version / Amendment | Type B (Programs) | Tracks structural evolution (e.g., criteria changes due to laws). |
| Program Variant / Implementation | Type B (Programs) | Child programs for geographic rollouts (e.g., State-level adaptation). |
| Successor Program | Type B (Programs) | Linkage ensuring continuity when a program ends and another begins. |
| Thematic Area / Sector | Type B (Programs) | E.g., WASH, Education, Nutrition. |
| Eligibility Criteria | Type B (Programs) | Rules governing enrollment. |
| Eligibility Version | Type B (Programs) | Binds an enrollment to the rules active at that time. |
| Mutual Exclusivity Rule | Type B (Programs) | Prevents double-dipping across conflicting programs. |
| Prerequisite Rule | Type B (Programs) | E.g., Must complete Program A to enter Program B. |
| Intervention Catalogue | Type B (Programs) | The menu of available support options (recurring, phased, conditional). |
| Milestone / Conditionality | Type B (Programs) | E.g., Cash transfer conditional on school attendance. |
| Program Cycle / Phase | Type B (Programs) | Temporal bounding (Launch, Active, Suspended, Closed). |
| Funding Constraint | Type B (Programs) | Earmarked vs Unrestricted, co-funding rules. |
| Compliance / Audit Checkpoint | Type B (Programs) | Mandatory reporting periods and regulator constraints. |
| Enrollment Status | Type B (Programs) | Waitlisted, Enrolled, Suspended, Graduated, Terminated. |
| Enrollment Prioritization Rule | Type B (Programs) | How waitlists are ordered (e.g., vulnerability score). |
| Humanitarian Override | Type B (Programs) | Formal exception mechanism bypassing criteria for severe cases. |
| Appeal Pathway | Type B (Programs) | Beneficiary mechanism to challenge an enrollment rejection. |
| Capacity / Quota | Type B (Programs) | Conceptual limits on program size. |
| Case Plan | Type C (Case Management) | Case Management orchestrates the individual's path through the program. |
| Support Delivery | Type C (Support Delivery) | SD executes the actual interventions. |
| Financial Ledger | Type D (Future / Out of Scope) | Detailed accounting is an ERP concern. |
